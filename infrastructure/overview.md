# 整體架構介紹

* 以 Parse 服務作為主要資料儲存以及各式伺服器邏輯之實踐
* 以 Firebase 作為即時性、原子性服務

「圖：架構」

### 為什麼要使用「雙後端」服務？

這樣的選擇有其歷史原因，早期 Parse 提供了純後端服務，而 Firebase 則是即時資料庫服務；時至今日，雖然 Parse 與 Firebase 都已發展成具有完整後端服務的平台，但早期在服務上還是有些微差異或不足

「圖：Parse 與 Firebase 服務差異」

所以考量多樣化方式取用資料、便於管理與呈現資料、更彈性的權限管控考量下，將資料儲存與伺服器邏輯實踐於 Parse 平台；而為了取用更完整的即時通訊能力、更完整支援的交易處理能力，將即時服務實踐在 Firebase 平台
